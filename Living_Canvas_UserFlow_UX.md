# Living Canvas
## User Flow и UX-сценарии

---

## 1. Общая логика пользовательского опыта

Пользователь не «управляет задачами».  
Пользователь **размещает мысли в пространстве** и наблюдает, как они связываются, уточняются и выходят на первый план.

UX строится вокруг:
- пространства (Canvas)
- внимания
- постепенного уточнения

---

## 2. Базовые пользовательские роли

### 2.1 Основная персона
- Knowledge worker
- 3–10 активных проектов
- Высокая когнитивная нагрузка
- Негативное отношение к спискам и Kanban

---

## 3. High-level User Flow

### Flow A: Первый вход (Cold Start)

1. Пользователь открывает приложение
2. Видит пустой Canvas с мягкой подсказкой:
   «Двойной клик — чтобы зафиксировать мысль»
3. Создаёт первый Node
4. Node автоматически центрируется
5. Система предлагает:
   - добавить связь
   - раскрыть заметку
   - оставить как есть

Цель: снять страх «пустоты»

---

### Flow B: Фиксация мысли

1. Пользователь двойным кликом создаёт Node
2. Вводит короткий текст
3. Node появляется в Card View
4. Пользователь:
   - перетаскивает Node
   - или сразу создаёт следующий

Важно: ни одного вопроса про «тип задачи»

---

### Flow C: Уточнение задачи

1. Пользователь приближает Node
2. Node переходит в Note View
3. Пользователь:
   - дописывает детали
   - выделяет фрагмент текста
   - превращает его в новый Node
4. Между Node создаётся связь `explains`

---

### Flow D: Построение структуры

1. Пользователь отдаляет Canvas
2. Видит граф связей
3. Перетаскивает Node, группируя их визуально
4. Создаёт связи drag-to-connect
5. Выбирает тип связи

Группировка может быть:
- логической
- временной
- эмоциональной

---

### Flow E: Фокусировка («Сейчас»)

1. Пользователь активирует режим «Сейчас»
2. Canvas перестраивается
3. 3–5 Node выводятся в центр
4. Остальные уходят в тень
5. Пользователь работает только с активными Node

---

## 4. UX-сценарии

### Сценарий 1: Работа с новым проектом

**Контекст:** пользователь начинает новый проект

1. Создаёт 5–7 Node с хаотичными идеями
2. Раскидывает их по Canvas
3. Начинает соединять связями
4. Масштабирует Canvas для понимания структуры
5. Фиксирует 2–3 Node как ключевые

Результат: появляется «карта проекта»

---

### Сценарий 2: Разгрузка головы

**Контекст:** пользователь перегружен

1. Быстро создаёт множество Node без структуры
2. Не связывает их сразу
3. Включает режим «Сейчас»
4. Система предлагает наиболее связанные Node
5. Пользователь начинает с них

Результат: тревога снижается за счёт внешней структуры

---

### Сценарий 3: Исследовательская работа

**Контекст:** аналитика, R&D, дизайн

1. Пользователь ведёт длинные заметки
2. Выделяет гипотезы
3. Превращает их в Node
4. Связывает с источниками и выводами
5. Использует масштаб для перехода от деталей к целому

---

### Сценарий 4: Возврат спустя время

**Контекст:** пользователь не заходил несколько дней

1. Открывает Canvas
2. Видит граф, а не список просрочек
3. Масштабируется к активной области
4. Включает «Сейчас»
5. Быстро восстанавливает контекст

---

## 5. UX-анти-паттерны (запрещено)

- Обязательные формы при создании Node
- Модальные окна без крайней необходимости
- Требование выбора типа элемента
- Навязывание workflow

---

## 6. Критерии UX-успеха

Интерфейс считается успешным, если:
- пользователь может начать работу без обучения
- мысли фиксируются быстрее, чем в текстовом редакторе
- структура возникает органически
- Canvas заменяет списки и папки

---

## 7. Ключевая UX-идея (для агентов)

> Пользователь не управляет системой.  
> Система аккуратно **подсвечивает то, что уже происходит в голове пользователя**.

# Техническое задание
## Интерфейс инновационного трекера задач и планировщика
### Концепция: Living Canvas

## 1. Цель системы
Разработать интерфейс планировщика задач нового типа, объединяющего графы зависимостей, карточки задач и текстовые заметки в едином интерактивном пространстве.

Система должна отражать процесс мышления пользователя, а не навязывать методологию управления задачами.

## 2. Ключевые принципы
1. Единая сущность — Node  
2. Масштаб = контекст  
3. Связи важнее списков  
4. Фокус вместо перегруза  

## 3. Базовая модель данных

### 3.1 Node
- id  
- title  
- content  
- type_hint  
- clarity_level (0–1)  
- energy_cost (low/medium/high)  
- next_action  
- created_at  
- updated_at  

### 3.2 Edge
- source_node_id  
- target_node_id  
- relation_type  

Поддерживаемые типы:
- depends_on
- explains
- produces
- requires
- related
- blocks
- emotional_irritation

## 4. Canvas
- Бесконечный 2D-холст
- Pan / Zoom
- Свободное размещение Node
- Сохранение координат

## 5. Масштабные уровни

### 5.1 Graph View
- Минимальные узлы
- Видны связи и заголовки

### 5.2 Card View
- Карточки с ключевыми параметрами
- Перемещение и базовое редактирование

### 5.3 Note View
- Полноценный текстовый редактор
- Преобразование текста в Node

## 6. Поведение
- Drag & Connect
- Создание Node из текста
- Автосвязи между элементами

## 7. Режим «Сейчас»
- 3–5 активных Node
- Остальные затемняются
- Автоотбор по логике важности

## 8. UI-требования
- Без явных режимов
- Плавные анимации
- Минимум панелей

## 9. Ограничения
- Без Kanban
- Без обязательных статусов
- Без методологий

## 10. Критерии успеха
- Работа без списков
- Мышление вместо таск-трекинга
- Навигация через масштаб

## 11. Результат
Интерактивный прототип интерфейса без backend.
